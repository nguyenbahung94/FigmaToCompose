# Figma-to-Compose Pipeline

---

## Core Philosophy

**This is NOT a UI generator. It is a Design → System → Code pipeline.**

```
Figma             → input, NOT source of truth
Design System     → source of truth (component-mapping.json + token-mapping.json)
Generated Code    → output constrained by the system, not by Figma directly
```

If Figma uses a color not in `token-mapping.json` → FAIL (never emit hardcoded values).
If Figma uses a component not in `component-mapping.json` → FallbackLevel determines behavior.
Figma is never allowed to override the Design System. The system wins.

---

## Project Config Location

Before using any command, create your project config at:
```
<your-project>/
  .claude/figma-to-compose/
    config.json                     ← platform, typography_class, colors_source, etc.
    mappings/
      token-mapping.json            ← generated by init, customized by you
      component-mapping.json        ← generated by bootstrap, confirmed by you
```

The plugin reads project-local config first. Run `/figma-to-compose init` to generate these files automatically.

## Command Syntax

```
# Setup (run in order)
/figma-to-compose init              ← first-time setup: generate config + token-mapping
/figma-to-compose bootstrap         ← scan codebase, seed component-mapping (requires init)
/figma-to-compose validate          ← verify config + mappings are valid before use

# Debug
/figma-to-compose inspect <url>     ← show IR tree + token resolution + mapping (no codegen)
/figma-to-compose <url> --layout-map  ← spatial ASCII tree only (Stage 1, no IR/codegen)
/figma-to-compose review-mappings   ← review + confirm pending component mappings
/figma-to-compose suggest-mapping   ← list unmapped Figma components with suggestions
/figma-to-compose cleanup-orphans   ← list orphan generated blocks (Figma node removed)

# Execution
/figma-to-compose run <url>         ← smart: respects confidence gate, may auto dry-run
/figma-to-compose generate <url>    ← dumb: always generates, no confidence decision logic
```

**Flags (for `run` / `generate` only — not compatible with `--layout-map`):**
- `--dry-run`: Full pipeline, print IR + mapping, do NOT write files
- `--allow-write-low-confidence`: Bypass auto dry-run gate (avg_confidence < 0.70) ONLY.
  Does NOT bypass: unresolved tokens, Major fallback blocks.
- `--verbose`: Show INFO logs (default: WARN + ERROR only)

**`--layout-map`:** Replaces pipeline entirely — Stage 1 only, spatial ASCII tree output. MUTUALLY EXCLUSIVE with all flags.

**Auto dry-run gate (`run` only):** IF avg_confidence < 0.70 → switch to dry-run automatically. Pass `--allow-write-low-confidence` to override.

**Logging levels:**
- `INFO` → pipeline stage transitions (verbose only)
- `WARN` → fallback used, approximate token match, stale mapping
- `ERROR` → hard fail — unresolved token, missing required slot, parse error

## Bootstrap Procedure

Run once: `/figma-to-compose bootstrap`

### Step 1 — Extract Compose components from codebase

```
Run: grep -r "@Composable" <compose_source_dir> --include="*.kt" -l
     (compose_source_dir = resolved from config.json "compose_package" → e.g. src/main/java/)
For each file found:
  Extract: function name, full parameter list, parameter types
  Store: { name, signature, file_path }
```

**Component filter rules (applied after extraction):**
```
INCLUDE:
  - public or internal visibility
  - Has @Composable annotation
  - Has ≥ 2 parameters
  - Name does NOT end with "Preview"

EXCLUDE:
  - @Preview annotated functions
  - private functions
  - Functions with < 2 params (likely helpers or wrappers)
  - Functions in files named *Preview*.kt
```

**AI matching score (Step 3):**
```
score = 0.5 * name_similarity + 0.3 * param_similarity + 0.2 * usage_frequency
Accept ONLY if score ≥ 0.70. Otherwise → mark as "unmapped", needs_review: true.
NEVER accept a candidate present in the entry's rejected_candidates list (score forced = 0).
```

### Step 2 — Get Figma component suggestions

```
Call: get_code_connect_suggestions with the Figma file key from the URL
Returns: list of Figma component names + suggested code matches
```

### Step 3 — AI match Figma names → Compose signatures

```
For each Figma component name:
  1. Exact name match against extracted Compose functions → confidence=0.95
  2. Fuzzy name match (edit distance ≤ 2) → confidence=0.80
  3. Semantic match (AI judgment) → confidence=0.70
  All stored with source=bootstrap
  DO NOT set confidence=1.0 — user review required first
```

### Step 4 — Show mapping table for one-time review

```
Display all bootstrap entries:
  | Figma Name     | → | Compose Component | Confidence | Confirm? |
  |----------------|---|------------------|------------|----------|
  | Card/Product     | → | ProductCard        | 0.85       | y/n      |
  | Button/Primary | → | AppButton        | 0.95       | y/n      |

User confirms each entry:
  y → confidence=1.0, source=manual → saved to component-mapping.json
  n → entry dropped
  skip → stays at bootstrap confidence, saved with source=bootstrap

Log: "Bootstrap complete. X entries confirmed, Y dropped, Z pending review."
```

## Init Procedure

Run once per project: `/figma-to-compose init`

**Pre-check:** If `.claude/figma-to-compose/config.json` already exists → ask user:
"Config found. Re-run init? This will overwrite config.json but preserve existing mappings. (y/n)"
If n → abort.

### Step 1 — Detect platform

```
Search strategy (in priority order):
  1. Find ANY file with @Composable annotation:
     grep -r "androidx.compose\|@Composable" . --include="*.kt" -l
  2. If multiple modules found → count @Composable functions per module:
     grep -r "@Composable" <module>/src --include="*.kt" -c
     → pick module with highest count
  3. Find ANY file with SwiftUI:
     grep -r "import SwiftUI" . --include="*.swift" -l
  4. If both Android + iOS found → ask user which platform to target
  5. Nothing found → ask user: "Which platform? (android/ios)"

Store detected platform + source module path for Steps 2-5.
```

### Step 2 — Scan color tokens (Android)

```
Source priority (try each, merge results):
  1. Parse colors_source (ask user for path, default: src/main/res/values/colors.xml):
     grep '<color name="' <colors_source>
     Extract: name="xxx" → #HEX
     → entry: { "hex": "#HEX", "resource": "xxx", "compose": "colorResource(R.color.xxx)", "needs_review": false }

  2. Search for Color() definitions in Kotlin:
     grep -r 'Color(0x' . --include="*.kt" | head -50
     For each unique hex NOT already in step 1 results:
       → add entry with "needs_review": true

  3. If < 3 colors found total → ask user:
     "Found only X colors. Please point me to your colors file or theme class."

Output: list of color token candidates
```

### Step 3 — Scan typography tokens (Android)

```
Source priority:
  1. Search for TextStyle definitions:
     grep -r 'TextStyle(' . --include="*.kt" -l
     For each file found → extract fontSize, fontWeight, lineHeight
     → map to typography.* key by size+weight heuristic:
       fontWeight=700 + fontSize≥20 → "typography.display-bold"
       fontWeight=700 + fontSize=18 → "typography.heading-bold"
       fontWeight=400 + fontSize=16 → "typography.body-regular"
       fontWeight=500 + fontSize=16 → "typography.body-medium"
       fontWeight=700 + fontSize=16 → "typography.body-bold"
       fontWeight=400 + fontSize=14 → "typography.subhead-regular"
       fontWeight=700 + fontSize=14 → "typography.subhead-bold"
       fontWeight=400 + fontSize=12 → "typography.caption-regular"
       Unrecognized combo → "typography.unknown-NNpx-WWw", needs_review: true

  2. Ask user: "Found X typography styles. Is there a typography class/object to use?
     Enter class name (e.g. AppTypography) or press Enter to skip:"
     If provided → store as typography_class in config

  3. If no TextStyle found → ask user to select typography file manually

Output: list of typography token candidates
```

### Step 4 — Scan spacing tokens (Android)

```
Source priority:
  1. Parse dimens.xml if exists:
     grep '<dimen' src/main/res/values/dimens.xml 2>/dev/null
     Extract dp values
  2. Search for .dp constants:
     grep -r '= [0-9]*\.dp\b' . --include="*.kt" | grep -o '[0-9]*\.dp' | sort -u | head -20
     Extract unique integer dp values

  Map to spacing.* keys:
    4dp  → spacing.xsmall
    8dp  → spacing.small
    16dp → spacing.medium
    24dp → spacing.large
    32dp → spacing.xlarge
    48dp → spacing.xxlarge
    Other → spacing.custom-Ndp, needs_review: true

Output: list of spacing token candidates
```

### Step 5 — Generate files

```
Write .claude/figma-to-compose/config.json:
{
  "platform": "<detected>",
  "schema_version": "1.0",
  "mapping_version": "1.0",
  "plugin_version": "1.x",
  "autonomy_phase": 1,
  "colors_source": "<detected_or_asked>",
  "typography_class": "<detected_or_asked_or_empty>",
  "compose_package": "<detected_or_asked>"
}

Write .claude/figma-to-compose/mappings/token-mapping.json:
  Use templates/android/token-mapping.template.json as base schema.
  Merge all color/typography/spacing candidates from Steps 2-4.
  Mark uncertain entries with "needs_review": true.

DO NOT write component-mapping.json — that is bootstrap's job.
```

### Step 6 — Summary

```
Print:
✓ Config written to .claude/figma-to-compose/config.json
✓ X color tokens extracted (Y need review)
✓ Z typography styles mapped (W need review)
✓ S spacing tokens extracted

⚠ Next steps (run in order):
  1. /figma-to-compose bootstrap   — scan @Composable functions → component mapping
  2. /figma-to-compose validate    — verify everything before first use
  3. /figma-to-compose review-mappings  — confirm pending entries
```

## Stage 1: Fetch + Normalize

**Call:** `get_design_context(nodeId, fileKey)` — call ONCE per URL, cache result for session.

**URL parsing:**
- `figma.com/design/<fileKey>/<name>?node-id=<nodeId>` → convert `-` to `:` in nodeId
- `figma.com/design/<fileKey>/branch/<branchKey>/...` → use branchKey as fileKey

**Normalize the node tree:**

```
Strip:
  - Hidden layers (visible=false)
  - Locked layers with no children
  - Junk groups: frame named "Group" or "Frame N" with no semantic meaning and exactly 1 child

Flatten:
  - Single-child wrapper frames with no fill, no stroke, no effect → inline the child
  - Redundant nesting: Frame → Frame → Component → unwrap outer frames

Preserve:
  - All named frames with semantic names (Header, Content, Footer, Actions, Body)
  - All component instances
  - All text nodes
  - Auto-layout direction (vertical/horizontal) and spacing
  - All fill, stroke, padding values
  - Figma node IDs (required for stable ID computation in Stage 5)
```

---

## Stage 2: Token Extract

**Token key format (strictly enforced):**
Regex: `^(spacing|color|typography)\.[a-z][a-z0-9-]*$`
Any key not matching → FAIL with parse error. No exceptions.

### Colors

token-mapping.json schema for colors:
```json
"color.primary": { "hex": "#0066CC", "compose": "MaterialTheme.colorScheme.primary" }
```

```
For each fill, stroke, text color from Figma:
  1. Exact match in token-mapping.json: find entry where token.hex == figma_hex (case-insensitive)
     → found: emit token.compose. DONE.

  2. Not found → grep colors.xml directly:
     grep -i "<color name=..." <colors_source>
     (colors_source = resolved from config.json "colors_source")
     Search for hex value (also resolve @color references recursively)
     → found as "color_name": emit colorResource(R.color.color_name)
                              + auto-append to token-mapping.json with source=auto
                              + log INFO: "Auto-added color.xxx from colors.xml"
     → DONE (no user action needed)

  3. Not in token-mapping.json AND not in colors.xml → CIELAB closest match:
     - ΔE < 2.0 → WARN "approximate match ΔE=X", use closest token
     - ΔE ≥ 2.0 → FAIL, report to user:
         "Color #XXXXXX not in colors.xml or token-mapping.json.
          Option A: Add to colors.xml → re-run (pipeline will auto-pickup)
          Option B: Map manually in token-mapping.json"

NEVER emit Color(0xFF...) or hardcoded hex. FAIL instead.
```

### Spacing

token-mapping.json schema for spacing:
```json
"spacing.medium": { "dp": 16, "compose": "16.dp" }
```

```
For each gap, padding value (in dp):
  1. Exact match: find entry where token.dp == figma_dp
     → found: emit token.compose
  2. No exact match → find candidates where abs(figma_dp - token.dp) ≤ 4
  3. One candidate → WARN "approximate match Δ=Xdp", use token
  4. Multiple candidates with equal Δ → auto-pick smallest token.dp (conservative sizing)
       WARN: "Spacing tie: 14dp → spacing.small (Δ=2) selected over spacing.medium (Δ=2)."
  5. No candidates within 4dp → FAIL
```

### Typography

```
For each text node:
  Match by ALL three criteria (all must pass — partial match not accepted):
    1. fontSize exact match
    2. fontWeight exact match
    3. lineHeight within 10% of token lineHeight
  If any criterion fails → FAIL, report to user:
    "Typography mismatch: Figma uses fontSize=X, fontWeight=Y, lineHeight=Z.
     No token in token-mapping.json matches all three criteria.
     Option A: Add entry to token-mapping.json under 'typography.*' key.
     Option B: Identify the closest <typography_class>.* style and confirm manually."
```

**Hard rule:** Generator does not proceed if any token remains unresolved.

## Stage 3: Screen vs Component Detection

Apply rules in priority order (semantic beats size):

```
Rule 1 (semantic — navigation intent):
  IF node contains any of (evaluated on raw Figma node data, before IR is built):
    - A child with a Figma prototype link of type "navigate" or "overlay-close"
    - A child node named "Back", "Close", or "Dismiss" at the top level of the frame
    - Distinct named top-level sections: all of Header + Content + Footer present simultaneously
  → SCREEN
  Note: these map to ActionType.Navigate / ActionType.Dismiss in Stage 5 IR — but Rule 1
  operates on raw Figma data, not IR types.

Rule 2 (semantic — instance count):
  IF the Figma node is a component instance AND
     that component appears ≥ 2 times in the Figma file
  → COMPONENT

Rule 3 (size heuristic — last resort):
  IF node width ≥ 90% of device frame width (360dp baseline)
     AND node contains ≥ 3 distinct named child sections
  → SCREEN

Rule 4 (default):
  → COMPONENT
```

**Known edge cases:**
- Modal full-screen → has Dismiss action → Rule 1 → SCREEN ✓
- Reusable full-width card → instance count ≥ 2 → Rule 2 → COMPONENT ✓
- Nested large frame → fails Rule 1 + 2 → Rule 3 applies

**Output paths:**

SCREEN:
```
<compose_output_dir>/<featureName>/
  <FeatureName>Fragment.kt
  <FeatureName>ViewModel.kt
  <FeatureName>Screen.kt
```
(compose_output_dir = resolved from config.json "compose_package" → e.g. src/main/java/com/example/presentation/)

COMPONENT:
```
<compose_output_dir>/components/
  <ComponentName>.kt
```

Feature/component name = Figma node name in PascalCase (spaces, slashes removed).

## Stage 4: Component Detect + Mapping Lookup

For each component node in the normalized tree:

### Step 1 — Lookup in component-mapping.json

```
Match priority:
  1. Exact key: "Button/Primary/Large"
  2. Variant prefix: "Button/Primary" matches "Button/Primary/*" entries
  3. Base name: "Button" matches "Button/*" entries (lowest priority)

Multiple matches found:
  diff = abs(top_confidence - second_confidence)
  diff > 0.1 → use highest confidence automatically
  diff ≤ 0.1 → STAGED: show both options, require user selection
    → selected entry: confidence += 0.05, source=manual
  NEVER resolve ties silently.
```

### Step 2 — Stale entry check

```
confidence < 0.5 → flag stale, require re-confirmation before use
confidence < 0.3 → treat as not found (entry effectively expired)
```

### Step 3 — FallbackLevel classification (when not found)

```
BEFORE classifying as fallback — check Material3 default mapping:
  "Card/*"    → androidx.compose.material3.Card
  "Button/*"  → androidx.compose.material3.Button
  "List/*" or "ListItem/*" → androidx.compose.material3.ListItem
  "Dialog/*"  → androidx.compose.material3.AlertDialog
  "Sheet/*"   → androidx.compose.material3.ModalBottomSheet
  "TextField/*" → androidx.compose.material3.TextField
  "Switch/*"  → androidx.compose.material3.Switch
  "Checkbox/*"→ androidx.compose.material3.Checkbox

  IF match found → use M3 component directly (log WARN: "No custom mapping — using Material3 fallback")
  IF no M3 match → then classify as Fallback below

Major (blocks generation if root-level, no M3 match):
  Component name contains: Card, List, Item, Form, Button, Sheet, Dialog, Screen
  Any primary navigation or data-display component with no M3 equivalent

Minor (WARN + continue with Box, no M3 match):
  Component name contains: Icon, Illustration, Avatar, Badge, Chip, Divider
  Unknown components at depth ≥ 3 in the tree
```

### Step 4 — Root-level Major fallback = BLOCK (only when no M3 fallback available)

```
IF Fallback(level=Major) AND node is direct child of screen/top frame:
  → BLOCK: do not emit any code
  → ERROR: "Blocked: root-level '<FigmaName>' has no mapping and no M3 equivalent.
            Option A: create @Composable fun <SuggestedName>(...) first.
            Option B: add '<FigmaName>' → '<ComposeName>' to component-mapping.json.
            Then re-run."
  → STOP pipeline

IF Fallback(level=Major) AND node is nested (not root):
  → Downgrade to STAGED mode (even if one-shot gate passed)
  → Show at checkpoint: "Major fallback: '<FigmaName>' will generate as Box. Continue? y/n"
  → Require explicit confirm

IF Fallback(level=Minor):
  → Log WARN
  → Continue, emit Box placeholder with actionable comment
```

### Step 5 — Update usage stats on successful match

```
entry.last_used = ISO timestamp now
entry.usage_count += 1
```

## Stage 5: Layout IR Builder

Construct the LayoutNode IR from normalized nodes + mapping results.
Generator reads ONLY the IR — never raw Figma data.

### Node mapping rules

```
Figma auto-layout VERTICAL   → LayoutNode.Vertical(children, spacing, padding, width, height, alignment, weight=0f)
Figma auto-layout HORIZONTAL → LayoutNode.Horizontal(children, spacing, padding, width, height, alignment, weight=0f)
Figma absolute-positioned    → LayoutNode.Stack(children=[PositionedChild(node, zIndex, alignment, offset)])
Figma component (mapped)     → LayoutNode.Component(mappedComponent, props, slots, action)
Figma text node              → LayoutNode.Text(content, typography, color, action)
Figma unmapped node          → LayoutNode.Fallback(description, reason, level)
```

### SizeMode mapping

```
Figma "fill container" → SizeMode.Fill
Figma "hug contents"   → SizeMode.Wrap
Figma fixed size       → SizeMode.Fixed(token) — token MUST exist or FAIL
```

### Stack / overlay detection

```
IF any sibling nodes use absolute positioning (x/y coords, not auto-layout):
  → Wrap all siblings in LayoutNode.Stack
  → Each child becomes PositionedChild:
      zIndex = Figma layer order (higher = rendered later = on top)
      alignment = derive from x/y relative to parent bounds:
        x near right + y near top → AlignmentProps(horizontal=End, vertical=Top)
        x near left  + y near top → AlignmentProps(horizontal=Start, vertical=Top)
        centered → AlignmentProps(horizontal=CenterHorizontally, vertical=CenterVertically)
      offset = delta between alignment anchor and actual position (in dp)

Render algorithm: children.sortedBy { it.zIndex }.forEach { render(it) }
Later in sorted list = drawn on top. Ties → original list order.
```

### PropValue binding

```
Text from Figma text node (varies by instance) → NodeRef("data.<paramName>")
Text constant across all instances             → Literal("hardcoded string")
Text transform (uppercase etc.)               → Expression(ExpressionType.Uppercase, [NodeRef(...)])

ExpressionType whitelist (no others allowed):
  Uppercase, Lowercase, FormatDate, Concat

NodeRef path formal grammar:
  data namespace: data.<identifier>(.<identifier>)*
    valid:   "data.shiftId", "data.job.title"
    invalid: "data.shift[0]", "data.shiftId.0"
  node namespace: node.<identifier>(.<identifier>)* | node.<identifier>.children[<int>](...)*
    valid:   "node.header.title", "node.children[0].text"
    invalid: "node.children", "node.[0].text"
  Regex: ^(data|node)(\.[a-zA-Z][a-zA-Z0-9]*|\.children\[\d+\])+$
  FAIL if path does not match — no silent acceptance.
```

### Action binding

```
Figma prototype → nav link  → ActionType.Navigate + ActionHandler.NavController
  params: { "destination": Literal("R.id.targetFragment") }

Tap with no nav             → ActionType.Click + ActionHandler.Callback
  params: {}  (onClick lambda passed by caller)

Figma "Close" overlay       → ActionType.Dismiss + ActionHandler.ViewModelEvent
  params: { "event": Literal("OnDismiss") }
```

### Stable Node ID (for incremental generation)

```
Priority (highest → lowest):
  1. hash(figmaNodeId + semanticRole)   ← survives rename, survives reorder
  2. "<ParentName>.<slotName>.<childType>"  ← semantic path fallback
  3. "<ParentName>.children[<index>]"       ← last resort

semanticRole = component type + slot context (e.g. "header.text", "actions.button")

IF Figma node ID unchanged (designer renamed "Header" → "TopSection"):
  → ID still matches → incremental update, NOT duplicate block

IF Figma node ID changed (node deleted + recreated):
  → treat as new → append new block, old block becomes orphan
  → WARN: "Orphan block: <id> — Figma node ID changed. Review manually."
```

### IR Debug Output (staged mode only)

Print before proceeding to generator:
```
IR Tree:
  Vertical(spacing=spacing.medium, width=Fill, height=Wrap) [id=ProductCard]
  ├── Component(ProductCard) [confidence=0.92, id=ProductCard.root]
  │   ├── slot:header → Text("Featured Item", typography.headline, color.on-surface)
  │   └── slot:actions → Component(AppButton, variant=Primary, size=Large)
  └── Fallback("Badge/Status", Minor) [id=ProductCard.badge]
```

## Stage 6: Compose Generator

Read the IR. Emit Kotlin. Never read Figma data directly.

### Hard rules

```
1. IF component in component-mapping.json → MUST use it. No exceptions.
2. IF component NOT in mapping → apply FallbackLevel from Stage 4 Step 3:
   Minor fallback: emit Box with TODO comment per Fallback template.
   Major fallback at root: BLOCK generation entirely.
   NEVER create a new named @Composable function for an unmapped component.
3. Colors: ALWAYS emit the exact `compose` value from token-mapping.json.
   NEVER construct Color(0x...) or any color literal. token-mapping.json is the only valid source.
4. Typography: ALWAYS emit <typography_class>.* from config.json — NEVER MaterialTheme.typography.* or hardcoded fontSize/fontWeight
5. Spacing:    ALWAYS emit token.compose (e.g. 16.dp) — NEVER raw Int dp without a token
6. NEVER emit raw Box() except for LayoutNode.Fallback. Fallback MUST log reason.
7. NEVER add // generated by AI or // auto-generated comments.
8. Slot rules:
   required=true + node missing → FAIL
   required=false + node missing → skip silently
   multiple=true → generate forEach slot
```

### Output style (all generated code must follow)

```
1. Modifier chaining only: .fillMaxWidth().padding(...) NOT nested modifiers
2. Trailing commas on all multi-line call sites
3. Named parameters always — no positional args
4. Long slot lambdas → extract to private @Composable (never inline > 10 lines)
5. Max line length: 120 characters
6. @Suppress("FunctionNaming") on every PascalCase @Composable
7. @Suppress("UnusedPrivateMember") on every @Preview
8. private const val for any numeric literal (except 0, 1, -1)
```

### Modifier emission order (canonical)

```
1. size      (fillMaxWidth, width, height, size)
2. weight    (only inside weighted Row/Column)
3. padding   (outer → inner)

```

## Stage 7: Confidence Gate

### Compute metrics

```
coverage       = matched_components / total_detected_components
avg_confidence = sum(entry.confidence) / count(matched_entries)
```

### Mode selection

```
IF avg_confidence < 0.70 AND --allow-write-low-confidence not set:
  → AUTO DRY-RUN
  → Print IR + mapping decisions, do NOT write files
  → Log ERROR: "Auto dry-run: avg confidence {X} < 0.70.
               Review mappings, then re-run (or add --allow-write-low-confidence)."

ELSE IF coverage ≥ 0.90 AND avg_confidence ≥ 0.90:
  → ONE-SHOT: generate + write without checkpoints

ELSE:
  → STAGED: 4 checkpoints
```

### Staged checkpoints

```
[1] After Stage 1 (Normalize): show node tree → y=continue, n=abort
[2] After Stage 4 (Component Detect): show mapping table, confirm unknowns
[3] After Stage 5 (IR Build): show IR tree → y=continue, n=abort
[4] After Stage 6 (Generate): show code preview → y=write files, n=abort

At each checkpoint: e = edit mapping entry inline, then re-run stage
```

### Major fallback override

```
ANY Major fallback at root level → BLOCK (overrides all modes, Stage 4 enforces this)
ANY Major fallback at nested level → force STAGED even if one-shot gate passed
```

## Stage 8: Incremental Generation

Never overwrite developer-edited code.

### Region marker format

```kotlin
// <figma-generated id="ProductCard.header" hash="a3f9c2">
@Composable
private fun ProductCardHeader(title: String) { ... }
// </figma-generated>
```

**hash** = first 6 chars of SHA-256 of the serialized IR node subtree.
Changes when Figma node structure changes. Same hash = skip regeneration.

### On re-run (file exists with markers)

```
For each IR node to generate:
  1. Compute stableNodeId + hash
  2. Search file for: // <figma-generated id="<stableNodeId>"
  3. Found + hash matches → SKIP (no Figma change)
  4. Found + hash differs → regenerate that block only (replace between markers)
  5. Not found → append new block at end of file
     Log WARN: "New block appended: <id>. Check for orphaned old block."
  Code OUTSIDE markers → NEVER touched.
```

### On re-run (file exists WITHOUT markers)

```
→ STOP
→ WARN: "File <path> exists without figma-generated markers.
         Option A: add markers manually around generated sections.
         Option B: pass --allow-write-low-confidence to overwrite entire file (destructive — loses all manual edits)."
→ Do NOT write until user explicitly passes --allow-write-low-confidence
```

### On new file

```
→ Create full file
→ Package declaration + imports → OUTSIDE markers
→ All generated functions → INSIDE markers
```

## Stage 9: Mapping Lifecycle

Run at end of every successful pipeline execution.

### Confidence decay

```
For each entry in component-mapping.json:
  IF entry.last_used == this run → no decay (was used)
  ELSE:
    rate = 0.995 if entry.usage_count >= 10 else 0.98
    entry.confidence *= rate

  confidence < 0.5 → flag "stale": require re-confirmation before next use
  confidence < 0.3 → remove entry:
    Log WARN: "Removed stale: '<FigmaName>' → '<Component>' (confidence decayed to <X>)"
```

### Consistency boost

```
ONLY for source = "bootstrap" or source = "manual"
source = "ai" → NO boost until user explicitly confirms (sets source = "manual")

context_key = parentComponentName + ":" + slotName + ":" + componentVariant
// e.g. "ProductCard:actions:PrimaryButton" ≠ "OrderCard:actions:SecondaryButton"

IF context_key appears ≥ 3 times in entry.usage_context_history:
  entry.confidence = min(1.0, entry.confidence + 0.01)
  Log INFO: "Confidence boost: <FigmaName> → <X> (consistent in <context_key>)"
```

### Conflict resolution

```
IF 2+ entries match same Figma node:
  diff = abs(confidence_A - confidence_B)
  diff > 0.1 → use higher confidence automatically
  diff ≤ 0.1 → STAGED: show both, require user selection
    → chosen: confidence += 0.05, source = "manual"
  NEVER resolve ties silently.
```

### Auto-save rules

```
source = "manual"    → always save
source = "bootstrap" → always save (user reviewed at bootstrap)
source = "ai"        → save only if confidence ≥ 0.85 OR user explicitly confirms
```

Write updated `component-mapping.json` at end of run.

## Common Mistakes

**Hardcoding values** — Any `Color(0x...)`, raw dp, or font size = VIOLATION. Always resolve to token. If token missing → FAIL, don't guess.

**Skipping bootstrap** — Running without bootstrap means empty `component-mapping.json` → everything falls back → generation blocked. Always bootstrap first.

**Treating Figma as source of truth** — If Figma has a color not in design system → FAIL. Never emit "approximately correct" code. The mapping wins.

**Generating new @Composable for known component** — If `ProductCard` is in mapping, NEVER create a new `ProductCardV2`. Use the mapping.

**Writing outside markers on re-run** — Generator touches ONLY content between `// <figma-generated>` tags. Never touches code outside.

**Spacing ties** — if 2 tokens have equal Δ to the Figma value: auto-pick the one with the smaller `dp` value (conservative sizing), log WARN. Do NOT ask user — just warn and proceed. Example: Figma=12dp, candidates spacing.small=8dp (Δ=4) and spacing.medium=16dp (Δ=4) → pick spacing.small.
